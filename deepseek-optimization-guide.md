# DeepSeek 模型配置优化指南

## 🎯 优化目标
1. **最大化上下文利用率** - 从16k提升到64k
2. **优化成本效益** - 精确成本计算
3. **智能模型选择** - 基于任务类型自动选择
4. **改善响应质量** - 优化生成参数

## 🔧 已实施的优化

### 1. DeepSeek 模型配置升级
```json
"custom-api-deepseek-com/deepseek-chat": {
  "contextWindow": 64000,      // 从16k提升到64k (4倍)
  "maxTokens": 8192,           // 从4k提升到8k (2倍)
  "temperature": 0.7,          // 创造性平衡
  "topP": 0.9,                 // 核采样优化
  "frequencyPenalty": 0.1,     // 减少重复
  "presencePenalty": 0.1       // 鼓励多样性
}
```

### 2. 新增推理专用模型
```json
"custom-api-deepseek-com/deepseek-reasoner": {
  "contextWindow": 32000,      // 中等上下文
  "maxTokens": 4096,           // 精确推理
  "temperature": 0.3,          // 低温度，高确定性
  "reasoning": true            // 启用推理模式
}
```

### 3. 智能模型选择策略
- **主模型**: DeepSeek Chat (64k上下文)
- **推理任务**: DeepSeek Reasoner (32k上下文，低温度)
- **成本感知**: 优先使用成本最低的模型
- **上下文感知**: 根据对话长度自动选择
- **任务感知**: 根据任务类型优化参数

### 4. 上下文压缩优化
```json
"compaction": {
  "mode": "aggressive",        // 积极压缩
  "targetRatio": 0.7,          // 压缩到70%
  "preserveRecent": 10,        // 保留最近10条消息
  "preserveImportant": true    // 保留重要消息
}
```

## 💰 成本优化策略

"cost": {

## 🚀 性能提升预期

### 上下文利用率
- **之前**: 16k窗口，经常超限 (169%使用率)
- **之后**: 64k窗口，充足空间 (预计<50%使用率)

### 响应质量
- **温度调优**: 0.7适合创造性，0.3适合精确性
- **惩罚机制**: 减少重复，增加多样性
- **核采样**: 提高回答相关性

### 成本控制
- **精确计费**: 准确跟踪API使用成本
- **智能降级**: 长对话自动切换到成本更低模型
- **缓存优化**: 利用缓存减少重复计算

## 📊 使用建议

### 1. 日常对话
- 使用 `deepseek` 别名 (64k上下文)
- 适合：一般问答、文档分析、创意写作

### 2. 复杂推理
- 使用 `deepseek-reasoning` 别名 (32k上下文)
- 适合：数学计算、逻辑分析、代码调试

### 3. 长文档处理
- 自动切换到 `qwen` (128k上下文)
- 适合：长文章分析、多文件处理

### 4. 图像任务
- 自动使用 `gemini` (视觉模型)
- 适合：图片分析、图表理解

## 🔍 监控指标

### 需要关注的指标

3. **模型切换频率**: 不应过于频繁
4. **成本/响应**: 监控每响应平均成本

- 每周检查配置有效性

- 根据使用模式微调参数

## ⚠️ 注意事项

### 潜在问题
1. **上下文过大**: 64k窗口可能增加延迟
2. **成本波动**: 精确计费可能显示更高成本
3. **模型切换**: 频繁切换可能影响对话连贯性

### 解决方案
1. 设置上下文使用警告阈值 (80%)
2. 启用成本预算提醒
3. 优化模型切换策略
