### 技术细节记录

#### 有效的配置字段（OpenClaw schema支持）：
```json
{
  "id": "deepseek-chat",
  "name": "DeepSeek Chat (64k Context)",
  "reasoning": false,
  "input": ["text"],
  "cost": {
    "input": 0.0005,
    "output": 0.0015,
    "cacheRead": 0.0001,
    "cacheWrite": 0.0003
  },
  "contextWindow": 64000,
  "maxTokens": 8192
}
```

#### 不支持的配置字段（导致验证失败）：
- temperature
- topP
- frequencyPenalty
- presencePenalty
- 以及其他高级生成参数

#### 学习到的经验：
1. **OpenClaw配置schema限制**：只能使用预定义的字段
2. **配置验证重要性**：必须通过`openclaw doctor --fix`或schema验证
3. **增量修改策略**：直接编辑配置文件比使用config.patch更可靠
4. **备份的重要性**：修改前自动备份配置
