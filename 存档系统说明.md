# DeepSeek V3 + 存档系统 实施方案

## 📊 成本分析
- **DeepSeek V3**：输入 0.5元/百万tokens，输出 2元/百万tokens（最便宜）
- **上下文窗口**：16,000 tokens（偏小）
- **解决方案**：智能存档 + 按需加载

## 🗂️ 存档结构

### 1. 每日存档
```
memory/YYYY-MM-DD.md
```
- 记录当日所有对话摘要
- 保存重要决策、文件路径、用户偏好
- 文件大小控制在 2-3KB（约 500-800 tokens）

### 2. 长期记忆
```
MEMORY.md
```
- 重要信息提炼（用户习惯、技术配置等）
- 定期从每日存档中提取关键信息
- 保持简洁，只存真正需要长期记住的内容

### 3. 任务输出
```
workspace/任务名称_日期.txt
```
- 具体任务成果（如歌单、代码、配置等）
- 按需创建，独立于对话存档

## 🔄 工作流程

### 对话开始时：
1. 读取 `memory/最近3天.md`（摘要）
2. 读取 `MEMORY.md`（长期记忆）
3. 当前对话保持在新上下文窗口

### 对话过程中：
1. 重要信息实时保存到当日存档
2. 任务成果保存到独立文件
3. 定期检查上下文使用量

### 对话结束时：
1. 更新当日存档文件
2. 如有重要信息，更新 `MEMORY.md`
3. 清理临时上下文

## 📈 节省效果估算

假设：
- 每日对话：10,000 tokens
- 存档文件：2,000 tokens（压缩摘要）
- 实际加载：3,000 tokens（最新对话 + 存档摘要）

**节省**：7,000 tokens/天（约 70%）

按月计算：
- 原始：300,000 tokens
- 存档后：90,000 tokens
- 节省：210,000 tokens

**成本节省**：
- 输入：210k tokens × 0.5元/百万 = 0.105元/月
- 输出：假设1:3比例，再节省 0.315元/月
- **总计**：约 0.42元/月
