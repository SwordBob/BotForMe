# 🎯 第5章 信息系统工程 - 快速复习卡片

## 📌 核心考点速记（考前必看！）

### 1. 软件架构风格（⭐⭐⭐⭐⭐）
```
数据流风格 → 批处理、管道/过滤器
调用返回 → 主程序/子程序、面向对象、层次
独立构件 → 进程通信、事件驱动
虚拟机 → 解释器、基于规则
仓库 → 数据库、黑板、超文本
```

### 2. 架构评估（⭐⭐⭐⭐）
- **敏感点**：影响单个质量属性
- **权衡点**：影响多个质量属性
- **ATAM**：最常用，基于场景
- **场景三要素**：刺激→环境→响应

### 3. 测试覆盖标准（⭐⭐⭐⭐⭐）
```
语句 < 判定 < 条件 < 条件/判定 < 条件组合 < 路径
最弱                                    最强
```

### 4. 黑盒测试方法（⭐⭐⭐⭐）
- **等价类划分**：数据分类
- **边界值分析**：边界错误多
- **判定表**：多条件组合
- **因果图**：输入输出关系

### 5. 设计原则（⭐⭐⭐⭐）
```
S - 单一职责
O - 开闭（对扩展开放，对修改封闭）
L - 里氏替换
I - 接口隔离
D - 依赖倒置
```

### 6. CSMM成熟度等级（⭐⭐⭐⭐⭐）
```
1级 初始级 → 无序、靠个人
2级 项目规范级 → 项目级规范
3级 组织改进级 → 组织标准、稳定
4级 量化提升级 → 统计分析、量化
5级 创新引领级 → 创新、引领
```

### 7. 部署策略（⭐⭐⭐）
| 策略 | 特点 |
|------|------|
| **蓝绿部署** | 双环境切换，快速回滚 |
| **金丝雀部署** | 灰度发布，风险可控 |

### 8. 需求层次（⭐⭐⭐）
```
业务需求 → 用户需求 → 系统需求
```

### 9. 用例关系（⭐⭐⭐）
- **包含**：必须执行，提取公共行为
- **扩展**：条件执行，分支场景
- **泛化**：继承关系

### 10. 数据模型（⭐⭐⭐⭐）
```
概念模型（E-R图）→ 逻辑模型（关系表）→ 物理模型（实现）
```

**完整性约束**：
- 实体完整性：主键非空唯一
- 参照完整性：外键有效
- 用户定义完整性：业务规则

## 🎯 考试技巧

### 选择题
1. 看到"事件驱动"→独立构件风格
2. 看到"加密影响安全和性能"→权衡点
3. 看到"量化、统计"→CSMM 4级
4. 看到"组织标准"→CSMM 3级
5. 看到"边界"→边界值分析
6. 看到"不运行程序"→静态测试
7. 看到"内部逻辑"→白盒测试

### 案例分析
1. 需求问题→加强验证、引入原型
2. 架构问题→选择合适的架构风格
3. 测试问题→补充测试类型、增加覆盖率
4. 配置问题→建立基线、版本控制

### 论文写作
1. 架构设计→ATAM评估、风格选择
2. 需求工程→获取、分析、验证
3. 质量测试→测试策略、自动化
4. 过程改进→CSMM实施
