# 📚 软考高项 - 第10章 项目进度管理 详细知识点总结

## 📖 章节概述

**项目进度管理**是项目管理十大知识领域之一，也是软考高项考试的核心章节。本章主要讲解如何确保项目在规定时间内完成，涉及从活动定义到进度控制的完整过程。第10章内容覆盖教材第312-348页，是计算题的重点章节。

---

## 🎯 核心概念与重要性

### 1. 项目进度管理定义
- **目标**：确保项目在规定时间内完成
- **核心**：平衡时间、成本、质量、范围四大约束
- **关键输出**：进度基准、进度计划
- **本质**：对项目活动进行规划、排序、估算、制定和控制的过程

### 2. 进度管理的重要性
- **按时交付**：是项目成功的关键指标之一
- **成本控制**：进度延误通常导致成本增加
- **客户满意度**：按时交付提高客户信任和满意度
- **资源优化**：合理安排资源，避免浪费
- **风险控制**：提前识别进度风险并制定应对措施

### 3. 进度管理的基本原则
- **计划先行**：先有计划，后有执行
- **动态调整**：根据实际情况调整进度计划
- **全员参与**：所有项目成员都要了解进度要求
- **持续监控**：定期检查进度执行情况

## 🔄 项目进度管理六大过程

### 过程1：规划进度管理
**目的**：制定进度管理计划，为后续进度活动提供指南

**主要活动**：
- 确定进度计划编制方法
- 定义进度计量单位
- 确定进度控制临界值
- 制定进度报告格式

**工具与技术**：
- 专家判断
- 分析技术
- 会议

**输出**：
- **进度管理计划**：包含进度模型开发、准确度、计量单位、控制临界值等

### 过程2：定义活动
**目的**：识别为完成项目可交付成果所需的具体活动

**输入**：
- 范围基准（WBS、WBS词典）
- 事业环境因素
- 组织过程资产

**工具与技术**：
- **分解**：将WBS工作包分解为更小的活动
- **滚动式规划**：近细远粗，逐步细化
- 专家判断

**输出**：
- **活动清单**：所有活动的详细列表
- **活动属性**：活动的详细信息（ID、名称、描述、负责人等）
- **里程碑清单**：重要时点或事件

### 过程3：排列活动顺序
**目的**：识别和记录活动之间的逻辑关系

**依赖关系类型**：
1. **强制性依赖（硬逻辑）**：工作性质决定的，不可改变
2. **选择性依赖（软逻辑）**：基于最佳实践或偏好
3. **外部依赖**：项目与非项目活动之间的关系
4. **内部依赖**：项目内部活动之间的关系

**逻辑关系（前导图法PDM）**：
- **完成-开始（FS）**：最常见，A完成B才能开始
- **开始-开始（SS）**：A开始B才能开始
- **完成-完成（FF）**：A完成B才能完成
- **开始-完成（SF）**：少见，A开始B才能完成

**工具与技术**：
- **前导图法（PDM/单代号网络图）**：节点表示活动
- **箭线图法（ADM/双代号网络图）**：箭头表示活动
- 确定依赖关系
- 提前量与滞后量

**输出**：
- **项目进度网络图**：显示活动逻辑关系

### 过程4：估算活动持续时间
**目的**：估算完成单项活动所需的工作时段数

**估算方法**：

| 方法 | 特点 | 准确度 | 适用场景 |
|------|------|--------|----------|
| **专家判断** | 基于经验 | 中 | 缺乏历史数据 |
| **类比估算** | 自上而下，快速 | 低 | 类似项目 |
| **参数估算** | 基于历史数据模型 | 中 | 有可靠参数 |
| **三点估算** ⭐⭐⭐ | 考虑不确定性 | 高 | 风险较大 |
| **储备分析** | 包含应急时间 | 中高 | 不确定性高 |

**三点估算（PERT技术）公式（必背）**：
```
期望时间 Te = (乐观时间O + 4×最可能时间M + 悲观时间P) ÷ 6

标准差 σ = (P - O) ÷ 6

方差 σ² = [(P - O) ÷ 6]²
```

**概率范围**：
- Te ± 1σ：68.26%的概率
- Te ± 2σ：95.46%的概率  
- Te ± 3σ：99.73%的概率

**输出**：
- **活动持续时间估算**：包含可能的变化范围
- **估算依据**：记录估算方法和假设

### 过程5：制定进度计划 ⭐⭐⭐⭐⭐ **考试重点**
**目的**：分析活动顺序、持续时间、资源需求和进度约束，创建进度模型

**关键路径法（CPM）核心概念**：
- **关键路径**：网络图中最长的路径，决定项目最短工期
- **关键活动**：关键路径上的活动，总浮动时间为0
- **浮动时间（时差）**：
  - **总浮动时间（TF）**：活动可以延误而不影响项目总工期的最大时间
  - **自由浮动时间（FF）**：活动可以延误而不影响后续活动最早开始的时间

**六个时间参数计算（必背）**：

| 参数 | 含义 | 计算公式 |
|------|------|----------|
| **ES** | 最早开始时间 | 正推法，取紧前活动EF的最大值 |
| **EF** | 最早结束时间 | EF = ES + 持续时间 |
| **LS** | 最晚开始时间 | LS = LF - 持续时间 |
| **LF** | 最晚结束时间 | 逆推法，取紧后活动LS的最小值 |
| **TF** | 总浮动时间 | TF = LS - ES = LF - EF |
| **FF** | 自由浮动时间 | FF = 紧后活动ES的最小值 - 本活动EF |

**计算步骤**：
1. **正推法（从左到右）**：计算ES、EF
2. **逆推法（从右到左）**：计算LS、LF  
3. **计算浮动时间**：TF、FF
4. **识别关键路径**：TF=0的活动连接成的路径

**资源优化技术**：

| 技术 | 目的 | 特点 | 影响 |
|------|------|------|------|
| **资源平衡** | 解决资源过度分配 | 调整活动时间 | 可能延长工期，改变关键路径 |
| **资源平滑** | 优化资源使用 | 在浮动时间内调整 | 不改变关键路径，不延长工期 |

**进度压缩技术**：

| 技术 | 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **赶工** | 增加资源 | 缩短工期 | 增加成本 | 有额外预算 |
| **快速跟进** | 并行活动 | 不增加成本 | 增加风险 | 有浮动时间，风险可控 |

**建模技术**：
- **假设情景分析**：评估不同情景对进度的影响
- **模拟（蒙特卡洛分析）**：评估项目在特定日期完成的概率

**输出**：
- **进度基准**：经批准的进度计划
- **项目进度计划**：甘特图、里程碑图等
- **进度数据**：支持进度计划的信息
- **项目日历**：工作时间和非工作时间

### 过程6：控制进度
**目的**：监督项目状态，管理进度基准变更

**绩效测量工具**：
- **挣值管理（EVM）**：
  - **进度偏差（SV）**：SV = EV - PV
    - SV > 0：进度超前
    - SV < 0：进度落后
  - **进度绩效指数（SPI）**：SPI = EV / PV
    - SPI > 1：效率高
    - SPI < 1：效率低
- **趋势分析**：预测未来绩效
- **关键路径法**：监控关键路径变化

**进度控制活动**：
- 比较实际进度与计划进度
- 分析偏差原因
- 采取纠正或预防措施
- 管理进度变更

**输出**：
- 工作绩效信息
- 进度预测
- 变更请求
- 更新项目管理计划

## 📊 重要图表与工具

### 1. 甘特图（横道图）
- **特点**：直观显示活动和时间关系
- **用途**：进度计划展示、进度跟踪
- **优点**：简单易懂，适合向管理层汇报

### 2. 里程碑图
- **特点**：只显示关键事件
- **用途**：高层级进度汇报
- **优点**：突出重点，简洁明了

### 3. 项目进度网络图
- **前导图（PDM）**：节点表示活动
- **箭线图（ADM）**：箭头表示活动
- **用途**：显示活动逻辑关系，计算关键路径

### 4. 时标网络图
- **特点**：结合网络图和甘特图
- **用途**：直观显示活动逻辑和时间关系

### 5. 进度比较横道图
- **特点**：对比计划进度和实际进度
- **用途**：进度控制，偏差分析

## 🧮 核心计算公式总结

### 1. 三点估算公式
```
期望时间 Te = (O + 4M + P) ÷ 6
标准差 σ = (P - O) ÷ 6
方差 σ² = [(P - O) ÷ 6]²
```

### 2. 关键路径计算
```
ES（最早开始）：前序活动EF的最大值
EF（最早完成）：ES + 持续时间
LS（最晚开始）：LF - 持续时间  
LF（最晚完成）：后续活动LS的最小值
TF（总浮动）：LS - ES 或 LF - EF
FF（自由浮动）：后续活动ES的最小值 - 本活动EF
```

### 3. 挣值管理公式
```
进度偏差 SV = EV - PV
进度绩效指数 SPI = EV / PV
```

### 4. 完工时间估算
```
完工尚需时间 ETC = (总工期 - 已完成工期) / SPI
完工估算时间 EAC = 当前时间 + ETC
```

## ⚠️ 常见问题与解决方案

### 问题1：进度估算不准确
**原因**：
- 缺乏历史数据
- 需求不明确
- 技术不确定性

**解决方案**：
- 使用三点估算考虑风险
- 建立组织过程资产库
- 采用渐进明细的规划方式

### 问题2：关键路径频繁变化
**原因**：
- 活动延迟
- 依赖关系变化
- 资源不可用

**解决方案**：
- 定期重新计算关键路径
- 建立进度缓冲
- 加强关键活动监控

### 问题3：资源冲突严重
**原因**：
- 资源过度分配
- 技能不匹配
- 资源可用性变化

**解决方案**：
- 实施资源平衡
- 培训多技能人员
- 建立资源池

### 问题4：进度基准频繁变更
**原因**：
- 范围蔓延
- 客户需求变化
- 风险发生

**解决方案**：
- 严格执行变更控制流程
- 加强范围管理
- 建立有效的沟通机制

## 🔗 与其他知识领域的关系

### 1. 与范围管理
- **WBS是定义活动的基础**
- 范围变更是进度变更的主要原因
- 进度计划需要与范围基准保持一致

### 2. 与成本管理
- **进度影响成本**（时间成本）
- **挣值管理**综合衡量进度和成本绩效
- 进度压缩通常增加成本

### 3. 与质量管理
- 赶工可能影响质量
- 进度压力可能导致质量妥协
- 需要在进度和质量之间平衡

### 4. 与资源管理
- 资源可用性决定进度可行性
- 进度计划指导资源分配
- 资源约束影响进度优化

### 5. 与风险管理
- 进度风险识别和应对
- 进度缓冲管理
- 蒙特卡洛分析评估进度风险

### 6. 与沟通管理
- 进度信息需要及时沟通
- 进度报告是重要沟通内容
- 进度变更需要通知相关干系人

## 📝 考试重点与答题技巧

### 必考知识点（⭐⭐⭐⭐⭐）
1. **关键路径计算**（前推法、后推法、浮动时间）
2. **三点估算和PERT**（公式、标准差、概率计算）
3. **进度压缩技术**（赶工vs快速跟进的区别）
4. **资源优化技术**（资源平衡vs资源平滑）
5. **挣值管理**（SV、SPI计算和解释）

### 常见考题类型
1. **计算题**（30-40%）
   - 关键路径计算
   - 三点估算和概率
   - 挣值管理计算

2. **概念题**（40-50%）
   - 工具技术区别
   - 过程输入输出
   - 最佳实践选择

3. **情景题**（20-30%）
   - 进度问题分析
   - 解决方案选择
   - 变更管理流程

### 答题技巧
1. **计算题步骤**：
   - 先画简单的网络图
   - 按步骤计算（正推→逆推→浮动→关键路径）
   - 检查计算结果是否合理

2. **概念题判断**：
   - 理解定义，不要死记硬背
   - 注意相似概念的细微差别
   - 结合实际项目管理经验

3. **情景题分析**：
   - 识别问题本质
   - 回忆相关工具技术
   - 选择最合适的解决方案
   - 考虑方案的副作用

4. **时间管理**：
   - 计算题先做，概念题后做
   - 难题标记，最后处理
   - 留出检查时间

## ✅ 本章总结要点

1. **进度管理是动态过程**：需要持续监控和调整，不是一次性的计划
2. **关键路径是核心**：决定项目最短工期，需要重点监控
3. **估算要考虑风险**：三点估算比单点估算更准确
4. **进度压缩有代价**：赶工增加成本，快速跟进增加风险
5. **资源约束影响进度**：需要平衡资源分配和进度要求
6. **进度基准是控制依据**：变更需要正式流程，不能随意修改
7. **挣值管理是重要工具**：综合衡量进度和成本绩效
8. **沟通是关键**：进度信息需要及时准确地传达给相关方
