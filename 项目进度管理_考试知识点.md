# 📚 项目进度管理 - 考试知识点总结

## 📖 章节概述
**项目进度管理**是项目管理十大知识领域之一，确保项目按时完成。本章涵盖从活动定义到进度控制的完整过程。

---

## 🎯 核心概念

### 1. 项目进度管理定义
- **目标**：确保项目在规定时间内完成
- **核心**：平衡时间、成本、质量、范围的关系
- **关键输出**：进度基准、进度计划

### 2. 进度管理的重要性
- 按时交付是项目成功的关键指标
- 进度延误会导致成本增加、客户满意度下降
- 有效的进度管理有助于资源优化和风险控制

---

## 🔄 进度管理过程组（6个过程）

### 过程1：规划进度管理
- **目的**：制定进度管理计划
- **工具**：专家判断、分析技术、会议
- **输出**：进度管理计划

### 过程2：定义活动
- **目的**：识别为完成项目可交付成果所需的具体活动
- **输入**：范围基准、WBS
- **工具**：分解、滚动式规划
- **输出**：活动清单、活动属性、里程碑清单

### 过程3：排列活动顺序
- **目的**：识别和记录活动之间的逻辑关系
- **工具**：
  - **前导图法（PDM/单代号网络图）**：4种依赖关系
    - 完成-开始（FS）
    - 开始-开始（SS）
    - 完成-完成（FF）
    - 开始-完成（SF）
  - **箭线图法（ADM/双代号网络图）**
  - **确定依赖关系**：强制、选择、外部、内部
- **输出**：项目进度网络图

### 过程4：估算活动持续时间
- **目的**：估算完成单项活动所需的工作时段数
- **工具**：
  - **专家判断**
  - **类比估算**（自上而下）
  - **参数估算**（使用历史数据）
  - **三点估算**（PERT技术）
    - 公式：`(乐观时间 + 4×最可能时间 + 悲观时间) ÷ 6`
    - 标准差：`(悲观时间 - 乐观时间) ÷ 6`
  - **储备分析**（应急储备、管理储备）
- **输出**：活动持续时间估算

### 过程5：制定进度计划
- **目的**：分析活动顺序、持续时间、资源需求和进度约束
- **工具**：
  - **关键路径法（CPM）**
    - 关键路径：总浮动时间为0或最小的路径
    - 总浮动时间（TF）：`LS - ES` 或 `LF - EF`
    - 自由浮动时间（FF）：不影响后续活动最早开始的时间
  - **关键链法（CCM）**
    - 考虑资源约束
    - 设置缓冲时间（项目缓冲、接驳缓冲）
  - **资源优化技术**
    - 资源平衡（Resource Leveling）
    - 资源平滑（Resource Smoothing）
  - **进度压缩技术**
    - **赶工**：增加资源，成本增加
    - **快速跟进**：并行活动，风险增加
  - **建模技术**
    - 假设情景分析
    - 模拟（蒙特卡洛分析）
- **输出**：
  - 进度基准
  - 项目进度计划（甘特图、里程碑图）
  - 进度数据
  - 项目日历

### 过程6：控制进度
- **目的**：监督项目状态，管理进度基准变更
- **工具**：
  - **绩效审查**
    - 趋势分析
    - 关键路径法
    - 挣值管理（EVM）
      - 进度偏差（SV）：`EV - PV`
      - 进度绩效指数（SPI）：`EV / PV`
      - SPI > 1：进度超前
      - SPI < 1：进度落后
  - **项目管理软件**
  - **资源优化**
  - **进度压缩**
- **输出**：
  - 工作绩效信息
  - 进度预测
  - 变更请求
  - 更新项目管理计划

## 📊 重要公式和计算

### 1. 三点估算（PERT）
- **期望持续时间（tₑ）**：`(O + 4M + P) ÷ 6`
- **标准差（σ）**：`(P - O) ÷ 6`
- **方差（σ²）**：`[(P - O) ÷ 6]²`
- **范围估算**：`tₑ ± nσ`（n=1:68%，n=2:95%，n=3:99.7%）

### 2. 关键路径计算
- **前推法**：
  - ES（最早开始）：前序活动EF的最大值
  - EF（最早完成）：`ES + 持续时间`
- **后推法**：
  - LF（最晚完成）：后续活动LS的最小值
  - LS（最晚开始）：`LF - 持续时间`
- **浮动时间**：
  - 总浮动时间：`LS - ES` 或 `LF - EF`
  - 自由浮动时间：`后续活动ES - 本活动EF`

### 3. 挣值管理（EVM）
- **计划价值（PV）**：计划完成工作的预算
- **挣值（EV）**：实际完成工作的预算
- **实际成本（AC）**：实际花费的成本
- **进度偏差（SV）**：`EV - PV`
- **进度绩效指数（SPI）**：`EV / PV`
- **完工尚需估算（ETC）**：
  - 典型偏差：`(BAC - EV) / CPI`
  - 非典型偏差：`BAC - EV`
- **完工估算（EAC）**：`AC + ETC`

## 🛠️ 工具和技术总结

### 进度计划工具
1. **甘特图**：显示活动和时间关系
2. **里程碑图**：显示关键事件
3. **网络图**：显示活动逻辑关系
4. **时标网络图**：结合甘特图和网络图

### 进度优化技术
1. **资源平衡**：解决资源过度分配
2. **资源平滑**：在浮动时间内调整资源
3. **进度压缩**：缩短工期
4. **关键链法**：考虑资源约束和缓冲

### 进度控制工具
1. **绩效测量**：EVM、趋势分析
2. **进度比较横道图**：实际vs计划
3. **项目管理软件**：MS Project、Primavera

## ⚠️ 常见问题和解决方案

### 问题1：进度延误
- **原因**：估算不准确、资源不足、范围蔓延
- **解决方案**：赶工、快速跟进、减少范围、增加资源

### 问题2：关键路径变化
- **原因**：活动延迟、依赖关系变化
- **解决方案**：重新计算关键路径、调整资源

### 问题3：资源冲突
- **原因**：资源过度分配、技能不匹配
- **解决方案**：资源平衡、资源平滑、培训

### 问题4：进度基准变更
- **原因**：范围变更、风险发生
- **解决方案**：变更控制流程、重新制定基准

## 🔗 与其他知识领域的关系

### 1. 与范围管理
- WBS是定义活动的基础
- 范围变更是进度变更的主要原因

### 2. 与成本管理
- 进度影响成本（时间成本）
- 挣值管理结合进度和成本

### 3. 与风险管理
- 进度风险识别和应对
- 进度缓冲管理

### 4. 与资源管理
- 资源可用性影响进度
- 进度计划指导资源分配

## 📝 考试重点提示

### 必考知识点
1. **关键路径计算**（前推法、后推法、浮动时间）
2. **三点估算和PERT**（公式、标准差、概率）
3. **进度压缩技术**（赶工vs快速跟进）
4. **挣值管理**（SV、SPI计算和解释）
5. **资源优化技术**（平衡vs平滑）

### 常见考题类型
1. **计算题**：关键路径、三点估算、EVM
2. **概念题**：工具技术区别、过程输入输出
3. **情景题**：进度问题分析和解决方案

### 答题技巧
1. **画图辅助**：网络图、甘特图
2. **公式记忆**：重点记忆核心公式
3. **理解概念**：不要死记硬背，理解原理
4. **时间管理**：计算题先做，概念题后做

## ✅ 总结要点

1. **进度管理是动态过程**，需要持续监控和调整
2. **关键路径是进度管理的核心**，决定项目最短工期
3. **进度压缩有代价**，赶工增加成本，快速跟进增加风险
4. **资源约束影响进度**，需要平衡和优化
5. **进度基准是控制的依据**，变更需要正式流程
6. **挣值管理是重要工具**，综合衡量进度和成本绩效
