# 查看隐藏文件指南

## 🔍 为什么文件被隐藏？

在 macOS/Unix 系统中，以点号（`.`）开头的文件是**隐藏文件**。这是标准做法，用于：

- **配置文件**：`.git`、`.obsidian`、`.openclaw`
- **系统文件**：`.DS_Store`、`.gitignore`
- **临时文件**：各种工具的缓存和状态文件

## 🚀 如何查看隐藏文件

### 方法1：终端查看（推荐）
```bash
# 查看当前目录所有文件（包括隐藏）
ls -la

# 只看隐藏文件
ls -la .*
```

### 方法2：Finder 临时显示
```bash
# 开启显示隐藏文件
defaults write com.apple.finder AppleShowAllFiles YES
killall Finder

# 关闭显示隐藏文件
defaults write com.apple.finder AppleShowAllFiles NO
killall Finder
```

### 方法3：Finder 快捷键
- 在 Finder 中按 `Cmd + Shift + .`（点号）切换显示隐藏文件

## 📁 重要隐藏文件说明

### 1. `.git/` - Git 版本控制
```
.git/
├── HEAD              # 当前分支
├── config           # Git 配置
├── objects/         # 文件内容存储
└── refs/           # 分支和标签引用
```
**作用**：Git 仓库的核心，不要删除或修改

### 2. `.obsidian/` - Obsidian 配置
```
.obsidian/
├── app.json         # 应用设置
├── core-plugins.json # 插件配置
├── editor.json      # 编辑器设置
└── git.json        # Git 插件配置
```
**作用**：Obsidian 笔记应用的配置

### 3. `.openclaw/` - OpenClaw 状态
```
.openclaw/
└── workspace-state.json  # 工作区状态
```
**作用**：OpenClaw 的工作区管理

### 4. `.gitignore` - Git 忽略规则
**作用**：告诉 Git 哪些文件不需要版本控制

### 5. `.DS_Store` - macOS 元数据
**作用**：Finder 的文件夹显示设置（可以安全删除）

## 🛠️ 实用命令

### 删除所有 `.DS_Store` 文件
```bash
find . -name ".DS_Store" -delete
```

### 创建配置文件的可见链接
```bash
# 创建符号链接，方便访问
ln -s .obsidian obsidian-config
ln -s .openclaw .openclaw-config
```

### 查看特定隐藏文件内容
```bash
# 查看 .gitignore
cat .gitignore

# 查看 Obsidian 配置
cat .obsidian/app.json

# 查看 OpenClaw 状态
cat .openclaw/workspace-state.json
```

## ⚠️ 注意事项

1. **不要随意删除**隐藏文件，除非你知道它们在做什么
2. **不要取消隐藏**配置文件，这可能导致工具异常
3. **备份重要文件**后再进行修改
4. **使用终端**管理隐藏文件更安全

## 🔧 最佳实践

### 对于开发者/高级用户
- 使用终端查看和管理隐藏文件
- 保持配置文件隐藏状态
- 定期清理不必要的隐藏文件（如 `.DS_Store`）

### 对于普通用户
- 使用 Finder 快捷键临时查看
- 不要修改隐藏文件
- 关注可见的文件内容

### 对于这个工作区
```bash
# 推荐的日常操作
ls -la              # 查看所有文件
rm .DS_Store        # 删除系统文件
ls .obsidian/*.json # 查看 Obsidian 配置
```
