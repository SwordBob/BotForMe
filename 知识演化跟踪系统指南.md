# 📈 知识演化跟踪系统指南

## 🎯 系统概述

这是一个完整的自动化系统，用于跟踪你的知识库演化，每周自动分析 Markdown 文件变化，更新关系图谱，并提供详细的维护报告。

## 📁 系统架构

```
📁 工作空间根目录/
├── 📁 scripts/                          # 核心脚本
│   ├── weekly-knowledge-evolution.py    # 演化分析引擎
│   ├── weekly-knowledge-maintenance.sh  # 主维护脚本
│   ├── create-obsidian-graph.py         # 关系图谱生成
│   └── analyze-md-relationships.py      # 基础分析
│
├── 📁 knowledge-maintenance-logs/       # 执行日志
├── 📁 knowledge-evolution-reports/      # 每周演化报告
├── 📁 knowledge-evolution-data/         # 历史快照数据
├── 📁 knowledge-backups/                # 自动备份
│
├── 📄 com.niejq.knowledge-maintenance.plist  # macOS 自动化配置
├── 📄 setup-knowledge-maintenance.sh         # 安装脚本
├── 📄 knowledge-maintenance-now.sh           # 手动执行脚本
├── 📄 check-maintenance-status.sh            # 状态检查脚本
└── 📄 uninstall-knowledge-maintenance.sh     # 卸载脚本
```

## 🚀 快速安装

### 方法1：一键安装
```bash
./setup-knowledge-maintenance.sh
```

### 方法2：手动安装步骤
1. **设置权限**：
   ```bash
   chmod +x scripts/*.sh
   chmod +x scripts/*.py
   ```

2. **创建目录**：
   ```bash
   mkdir -p knowledge-maintenance-logs \
            knowledge-evolution-reports \
            knowledge-evolution-data \
            knowledge-backups
   ```

3. **安装 LaunchAgent**：
   ```bash
   cp com.niejq.knowledge-maintenance.plist ~/Library/LaunchAgents/
   launchctl load ~/Library/LaunchAgents/com.niejq.knowledge-maintenance.plist
   ```

## ⏰ 执行计划

系统配置为 **每周日早上9点** 自动执行，包含以下任务：

### 每周自动执行流程
1. **📸 创建知识快照** - 记录当前文件状态
2. **📊 分析变化** - 对比上周数据
3. **📈 生成演化报告** - 详细的变化分析
4. **🔄 更新关系图谱** - 如果变化显著
5. **💾 数据备份** - 备份重要文件
6. **🧹 清理旧文件** - 按保留策略清理
7. **📋 生成维护摘要** - 执行结果总结

## 📊 输出文件

### 1. 演化报告 (`knowledge-evolution-reports/`)
每周生成一个详细的 Markdown 报告：
```

├── 📊 概览 - 关键指标
├── 🔄 变化分析 - 新增/修改/删除文件
├── 📁 目录结构 - 文件分布
├── 📈 历史趋势 - 文件数量变化
├── 🎯 维护建议 - 优化建议
└── 🔧 技术详情 - 系统状态
```

### 2. 执行日志 (`knowledge-maintenance-logs/`)
包含详细的执行记录：
```

maintenance-summary-2026-03-22.md # 维护摘要

```

### 3. 历史数据 (`knowledge-evolution-data/`)
保存12周的历史快照：
```

```

### 4. 自动备份 (`knowledge-backups/`)
每周备份重要文件：
```
knowledge-backup-2026-03-22.tar.gz
```

## 🔧 配置调整

### 1. 调整执行时间
编辑 `com.niejq.knowledge-maintenance.plist`：
```xml
<key>StartCalendarInterval</key>
<dict>
    <key>Weekday</key>   <!-- 0=周日, 1=周一, ..., 6=周六 -->
    <integer>0</integer>
    <key>Hour</key>      <!-- 0-23 -->
    <integer>9</integer>
    <key>Minute</key>    <!-- 0-59 -->
    <integer>0</integer>
</dict>
```

重新加载配置：
```bash
launchctl unload ~/Library/LaunchAgents/com.niejq.knowledge-maintenance.plist
launchctl load ~/Library/LaunchAgents/com.niejq.knowledge-maintenance.plist
```

### 2. 调整维护参数
编辑 `scripts/weekly-knowledge-maintenance.sh`：
```bash
# 配置部分
LOG_DIR="knowledge-maintenance-logs"
REPORT_DIR="knowledge-evolution-reports"
BACKUP_DIR="knowledge-backups"
WEEKLY_BACKUP=true
SEND_NOTIFICATION=false
```

### 3. 调整分析参数
编辑 `scripts/weekly-knowledge-evolution.py`：
```python
CONFIG = {
    "data_dir": "knowledge-evolution-data",
    "snapshot_file": "knowledge-snapshots.json",
    "report_dir": "knowledge-evolution-reports",
    "weeks_to_keep": 12,           # 保留12周数据
    "min_change_threshold": 0.01,  # 1% 变化阈值
}
```

## 📈 报告解读

### 关键指标说明

#### 1. 变化率
- **< 5%**: 稳定期，知识库变化不大
- **5-20%**: 活跃期，正在积极更新
- **> 20%**: 剧烈变化期，可能有重大重构

#### 2. 文件大小分布
- **小型文件 (<1KB)**: 可能是笔记片段，考虑合并
- **中型文件 (1-10KB)**: 标准笔记，易于管理
- **大型文件 (≥10KB)**: 可能是综合文档，考虑拆分

#### 3. 目录结构
- **根目录文件过多**: 建议创建子目录分类
- **大目录 (>20文件)**: 建议进一步细分
- **目录层次过深**: 考虑扁平化

### 维护建议类型

#### 1. 知识整理建议
- 合并相关的小文件
- 拆分过大的综合文档
- 重新组织目录结构

#### 2. 链接优化建议
- 为新增文件添加相关链接
- 检查孤立文件（无链接的文件）
- 增强重要文件之间的连接

#### 3. 内容质量建议
- 补充内容不足的文件
- 更新过时的信息
- 标准化格式和结构

## 🔍 故障排除

### 常见问题

#### 1. 系统未自动执行
```bash
# 检查服务状态
launchctl list | grep knowledge

# 重新加载服务
launchctl unload ~/Library/LaunchAgents/com.niejq.knowledge-maintenance.plist
launchctl load ~/Library/LaunchAgents/com.niejq.knowledge-maintenance.plist

# 查看系统日志
tail -f knowledge-maintenance-logs/launchd.log
```

#### 2. 脚本执行失败
```bash
# 检查权限
ls -la scripts/

# 测试脚本
./scripts/weekly-knowledge-maintenance.sh --help

# 查看详细错误
bash -x ./scripts/weekly-knowledge-maintenance.sh 2>&1 | head -50
```

#### 3. Python 依赖问题
```bash
# 安装依赖
pip3 install jieba

# 检查 Python 环境
python3 --version
python3 -c "import jieba; print('✅ jieba 正常')"
```

#### 4. 磁盘空间不足
```bash
# 检查磁盘使用
df -h .

# 清理旧文件
find knowledge-backups/ -name "*.tar.gz" -mtime +84 -delete
find knowledge-maintenance-logs/ -name "*.log" -mtime +28 -delete
```

### 日志分析

#### 错误日志位置
1. **系统服务日志**: `knowledge-maintenance-logs/launchd.log`
2. **每周执行日志**: `knowledge-maintenance-logs/weekly-YYYY-MM-DD.log`
3. **错误日志**: `knowledge-maintenance-logs/launchd-error.log`

#### 常见错误信息
- `Permission denied`: 权限问题，运行 `chmod +x scripts/*.sh`
- `ModuleNotFoundError`: Python 依赖问题，运行 `pip3 install jieba`
- `No such file or directory`: 路径问题，检查工作空间目录
